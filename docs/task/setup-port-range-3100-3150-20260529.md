# Setup Port Range 3100-3150

- Ngày: 29-05-2026
- Phạm vi: `setup.ps1`, `setup.sh`, `stop.ps1`, `stop.sh`, Docker Compose, `.env.example`, README và docs vận hành.

## Mục Tiêu

Chuyển runtime local sang dải port `3100-3150` và xác nhận chạy bằng `setup.ps1` không còn trả `502 Bad Gateway` qua nginx.

## Port Hiện Tại

| Service | Port |
| --- | --- |
| Web source | `3100` |
| API source | `3110` |
| nginx/tunnel | `3120` |
| PostgreSQL | `3132` |
| Qdrant HTTP/gRPC | `3133` / `3134` |
| Redis | `3139` |

## Thay Đổi Chính

- Cập nhật `.env.example` và `.env` local sang dải `3100-3150`.
- Cập nhật default port trong `setup.ps1`, `stop.ps1`, `setup.sh`, `stop.sh`.
- Cập nhật `infra/docker/docker-compose.yml` và root `docker-compose.yml` để external port dùng dải mới.
- Thêm validate port trong `setup.ps1`: các port runtime bắt buộc nằm trong `3100-3150`.
- Sửa `setup.ps1` để PowerShell không ngắt giả khi command ghi stderr hợp lệ, và chỉ bỏ qua lỗi Docker Compose non-zero khi hạ tầng thật sự reachable.
- Thêm kiểm tra nginx web proxy trong `setup.ps1`/`setup.sh` để bắt đúng lỗi `502` ở entry `/`.
- Cập nhật README và docs vận hành sang URL mới.

## Kết Quả

- `.\setup.ps1` chạy xong exit `0`.
- nginx `http://127.0.0.1:3120` trả `200`, không còn `502`.
- API trực tiếp `http://127.0.0.1:3110/health` trả `200`.
- Web trực tiếp `http://127.0.0.1:3100/agent-dashboard` trả `200`.
- API qua nginx `http://127.0.0.1:3120/api/v1/products` trả `200`.
